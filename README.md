# 地下菌丝网络

静态浏览器小游戏，直接打开 `index.html` 即可运行。

玩家在格子地图上扩张菌丝，连接树根、分解落叶、绕开干层，并处理微生物竞争。项目包含关卡数据结构、回合推进、资源计算、菌丝生长可视化、撤销和换关。

## 玩法说明

### 胜利条件
每个关卡有独立的胜利目标，在关卡数据中通过 `winCondition` 配置：
- `requiredTrees`：需要连接的树根数量
- `requiredLeaves`：需要分解的落叶数量

满足当前关卡的所有条件后，会在「土壤记录」中显示「网络稳定，森林进入共生状态。」的胜利提示。

### 关卡列表
- **松林边缘**：连接 3 处树根，分解至少 2 片落叶
- **桦树湿沟**：绕过干层连接 4 处树根（无需分解落叶）

### 资源规则
- 扩张菌丝消耗养分，不同土壤消耗不同：
  - 壤土：3 养分
  - 湿土：2 养分
  - 干层：6 养分
- 带微生物的格子额外消耗 3 养分
- 分解落叶获得 8 养分
- 连接树根时获得一次性养分回馈（湿土树根 5，其余 3），回合推进时每处已连接树根回馈 2 养分
- 每 8 格菌丝消耗 1 点维持费用

### 微生物扩散与可复现随机机制

微生物在回合推进时有 18% 概率向相邻空格扩散。为了支持策略复盘和撤销操作，游戏使用**基于关卡种子的确定性伪随机数生成器（PRNG）**，而非直接调用 `Math.random()`。

#### 机制说明

- **种子生成**：每个关卡在首次加载或重置时，根据关卡索引通过哈希公式生成固定的初始种子（`seedForLevel`），因此同一关卡每次重置都会得到相同的随机序列起点。
- **状态推进**：使用 Mulberry32 算法作为 PRNG，每次随机采样后将内部状态前移，更新后的值保存在关卡状态对象的 `_rngState` 字段中。
- **历史记录集成**：`saveHistory()` 通过 `JSON.stringify(state)` 将完整状态（包括 `_rngState`）序列化到历史栈，撤销时通过 `JSON.parse` 恢复，因此**撤销若干步后再次推进，会得到与原路径完全一致的扩散结果**。
- **关卡切换**：从关卡选择器切换或通过向导切换关卡时，`reset()` → `parseLevel()` 重新为新关卡初始化种子，序列互不干扰。

#### 对复盘的影响

1. **确定性路径**：在同一关卡下，若你从初始状态按完全相同的操作顺序（相同的生长选择 + 相同的回合推进时机）执行，微生物扩散的时间点和位置将完全一致。这意味着你可以对失败的策略进行精确复盘——只需重置关卡并重复操作步骤，就能验证哪里出了问题、调整哪一步可以避免损失。
2. **撤销不改变未来**：如果你推进 N 回合后撤销 M 回合，再重新推进 M 回合，扩散结果将与你从未撤销、直接推进 N 回合时完全相同。撤销不会"重新掷骰子"，不会让你通过反复撤销/推进来刷出更优的随机结果。
3. **换关后不保留**：切换到其他关卡再切回来，序列会重新初始化，因此两次独立游玩同一关卡的体验是一致的（不会因前一次游玩的进度而改变扩散）。
4. **策略建议**：由于扩散可复现，你可以将微生物视为"半确定"的事件——只要记住哪些回合会触发扩散，就能提前规划养分储备和菌丝走向。如果某一局因为微生物扩散导致养分透支，可记下是哪一步触发的，然后在更早的回合中预留养分或抢占微生物所在格（`competed` 后便不再消耗额外养分）。
